## 1. 이슈 개요 (Issue Overview)

액추에이터 1개 단독 연결 상태에서 물리 동작 테스트를 진행한 결과,
**`STATE_OPENING` 진입 직후 약 301ms 시점에 잘못된 MOTOR STALL 감지**가 발생하여
뚜껑이 열리지 못한 채 즉시 `STATE_EMERGENCY_STOP(6)`으로 전이되는 현상이 확인되었습니다.

```
[STATE] Transition: 1 -> 2
[DEBUG] Dist: 21.8 cm, Batt: 11.91 V, Curr: 3200.0 mA, State: 2
[EMERGENCY] MOTOR STALL DETECTED DURING OPEN! Current: 1085.9 mA (Limit: 800.0 mA)
[DIAGNOSTIC] Time Elapsed: 301 ms, Batt: 12.61 V, Dist: 23.4 cm
[SAFETY] All relays forced OFF and isolated (0mA standby).
[STATE] Transition: 2 -> 6
```

또한, `STATE_EMERGENCY_STOP(6)` 상태에서 수동으로 `resetEmergency()` API를 호출하지 않으면
시스템이 영구적으로 잠겨 자동 복귀가 불가능한 구조적 문제도 함께 확인되었습니다.

## 2. 원인 분석 (Root Cause Analysis)

### 2.1. Stall 한계값(800 mA) 과소 설정

| 전류 구간 | 예상값 (액추에이터 1개) | 비고 |
|-----------|----------------------|------|
| 무부하 No-load | ~350 mA | 기어박스 마찰 |
| 정격 부하 100N | ~800–1,000 mA | P = F × v = 6W / 0.65η |
| 기동 Inrush (0–500ms) | ~2,500–3,500 mA | 역기전력 부재 구간 |
| Stall (완전 잠김) | ~3,000–5,000 mA | 권선 저항 한계 |

- 기존 `currentStallLimit = 800 mA`는 **정격 부하 전류와 동일한 수준**으로,
  정상 가동 중에도 스톨로 오감지될 수밖에 없는 값이었습니다.

### 2.2. Inrush bypass 기간 부족 (300 ms)

- 기동 직후 300 ms 이내에만 스파이크를 무시하도록 설계되었으나,
  기계적 관성이 있는 구간에서도 높은 전류가 지속될 수 있어 bypass 구간이 짧았습니다.

### 2.3. 단일 샘플 즉시 판정 (노이즈 취약)

- INA219가 1회 측정에서 순간 스파이크를 반환할 경우 즉시 비상정지가 발동되어,
  실제 stall이 아닌 측정 노이즈에도 과민하게 반응하였습니다.

### 2.4. 비상정지 자동 복귀 기능 미구현

- `STATE_EMERGENCY_STOP`이 `forceAllRelaysOff()` 단일 호출로만 구성되어
  수동 API 호출(`resetEmergency()`) 없이는 영구 잠금 상태가 유지되었습니다.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. 액추에이터 2개 기준 전류 재산정

액추에이터 최종 구성: **12V / 100N / 60mm/s × 2병렬**

```
기계적 출력: P = 100N × 0.060 m/s = 6.0 W (단일)
전기 입력:   P_input = 6.0 / 0.65 = 9.2 W → I = 9.2 / 12 ≈ 770 mA (단일 정격)
2개 병렬:    정격 운전 최대 ~2,000 mA
Stall 기준:  2,000 × 1.5 = 3,000 mA (안전 마진 50%)
```

| 파라미터 | 기존 (1개 기준) | 변경 (2개 기준) |
|----------|---------------|---------------|
| `currentStallLimit` | 800 mA | **3,000 mA** |
| Inrush bypass | 300 ms | **500 ms** |
| 판정 방식 | 단일 샘플 즉시 | **연속 3회 (≈600 ms)** |

### 3.2. 연속 카운터 기반 Stall 판정 ([SmartBoxController.cpp](../src/SmartBoxController.cpp))

```cpp
// STATE_OPENING / STATE_CLOSING 공통 적용
static int openStallCount = 0;
if (hw.getMillis() - stateTimer > 500) {       // Inrush bypass 500ms
    if (motorCurrent > config.currentStallLimit) { // 3000 mA
        if (++openStallCount >= 3) {            // 연속 3회 초과
            // → EMERGENCY_STOP
        }
    } else {
        openStallCount = 0;                     // 정상 전류 → 카운터 리셋
    }
} else {
    openStallCount = 0;
}
```

- `[WARN] Stall candidate OPEN (n/3)` 로그로 중간 경고를 출력하여 디버깅 가시성 확보.

### 3.3. 비상정지 자동 복귀 기능 ([SmartBoxController.cpp](../src/SmartBoxController.cpp))

`STATE_EMERGENCY_STOP` 상태에서 설정 가능한 복귀 대기 시간(`emergencyRecoveryTime`, 기본 5초) 경과 후:

- 초음파 센서 감지 → `STATE_OPEN_START` 자동 전이
- 감지 없음 → `STATE_IDLE` 복귀

```
[RECOVERY] Emergency lock expired (5000 ms). Checking sensor...
  ├─ 사람 감지 → [RECOVERY] Human detected (22.3cm) → Auto-opening lid.
  └─ 없음     → [RECOVERY] No human detected → Returning to IDLE.
```

### 3.4. 설정 파라미터 추가 ([SmartBoxController.h](../src/SmartBoxController.h) / [ConfigManager.cpp](../src/ConfigManager.cpp))

`BoxConfig` 구조체에 `emergencyRecoveryTime = 5000` 필드 추가.
NVS Preferences 키 `"emrg_rec"` 로 영구 저장/로드 및 self-healing 기본값 적용.

## 6. 결론 및 향후 권장 사항 (Conclusion & Recommendations)

- **결론:** 액추에이터 스펙(12V 100N 60mm/s ×2)을 기반으로 전류 한계를 물리적으로 재산정하고, 노이즈 면역 연속 카운터 및 자동 복귀 로직을 도입하여 오작동 없는 안정적인 동작 환경을 구축했습니다.

- **향후 권장 사항:**
  1. **실부하 측정:** 두 액추에이터 동시 연결 후 시리얼 `[DEBUG] Curr:` 로그로 실제 정격 운전 전류를 측정하여 `currentStallLimit`을 현장 보정할 것을 권장합니다.
  2. **Web Dashboard 연동:** `emergencyRecoveryTime` 파라미터를 웹 대시보드 설정 UI에 노출하여 현장에서 즉시 조정 가능하도록 추가 개선을 권장합니다.
  3. **단위 테스트 업데이트:** `test_stall_current_detection` 테스트 케이스를 신규 한계값(3000 mA) 및 연속 카운터 로직에 맞게 갱신하여 회귀 방지를 유지할 것을 권장합니다.
