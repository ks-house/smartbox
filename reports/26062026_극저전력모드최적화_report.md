# 📄 보고서: 야간 극저전력 모드(Deep Power Diet) — CPU 동적 클럭 강하 및 적응형 센서 폴링 구현

- **작성일자:** 2026년 06월 20일
- **주제:** 야간 절전 모드 시 CPU 클럭을 80MHz로 강하하고, 초음파 센서 폴링 주기를 250ms로 완화하여 배터리 소모를 최소화하는 극저전력 최적화

---

## 1. 이슈 개요 (Issue Overview)
SmartBox의 `PowerManager`를 통해 야간 절전 모드(23:00~06:00)에서 Wi-Fi를 끄는 기능은 구현되어 있었으나, CPU 클럭(160MHz)과 센서 폴링 주기(50ms)는 주간과 동일하게 유지되어 불필요한 전력 소모가 발생하고 있었습니다.

야간에는 네트워크 통신이 필요 없고, 센서 반응 속도 요구도 낮으므로 시스템 자원 소모를 극한까지 줄일 여지가 있었습니다.

---

## 2. 원인 분석 (Root Cause Analysis)
- **CPU 클럭 고정:** ESP32-C6의 기본 동작 클럭이 160MHz로 고정되어 있어, 야간 모드에서도 불필요하게 높은 클럭으로 전류를 소모합니다. ESP32는 `setCpuFrequencyMhz()` API를 통해 80MHz/160MHz/240MHz 동적 전환을 지원하며, 80MHz에서도 GPIO 제어, 센서 읽기, 상태 머신 등 기본 로직은 완벽히 동작합니다.
- **센서 폴링 과다:** 50ms 주기의 초음파 센서 폴링은 야간에도 초당 20회 센서 읽기를 수행하여, 트리거 펄스 발생 및 `pulseIn()` 대기 시간 동안 CPU를 깨워 유지합니다. 야간에는 250ms(초당 4회)로도 충분한 반응성을 확보할 수 있습니다.

---

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. Dynamic CPU Frequency Scaling
- **[PowerManager.cpp](../src/PowerManager.cpp)**:
  - **야간 진입 시점:** Wi-Fi를 끈 직후 `setCpuFrequencyMhz(80)`을 호출하여 CPU 클럭을 80MHz로 강하합니다.
  - **주간 복귀 시점:** Wi-Fi를 켜기 직전 `setCpuFrequencyMhz(160)`을 호출하여 원래 클럭으로 복구합니다.
  - 두 호출 모두 `#ifndef NATIVE_BUILD` 가드로 보호하여 네이티브 테스트 빌드에서 컴파일 오류가 발생하지 않도록 했습니다.
  - 시리얼 로그: `[POWER] CPU Frequency scaled to 80 MHz` / `[POWER] CPU Frequency scaled to 160 MHz`

### 3.2. Adaptive Sensor Polling Rate
- **[SmartBoxController.cpp](../src/SmartBoxController.cpp)**:
  - 기존의 고정 50ms 폴링 주기를 삼항 연산자로 동적 변경:
    ```cpp
    unsigned long pollInterval = isSleep ? 250 : 50;
    ```
  - 주간(50ms): 초당 20회 폴링 → 빠른 반응성
  - 야간(250ms): 초당 4회 폴링 → 5배 전력 절감
  - 기존에는 야간 모드에서 센서 폴링을 완전 차단(`!isSleep && ...`)했으나, 야간에도 쓰레기통 기본 동작(뚜껑 개폐)이 유지되도록 설계를 변경했습니다.

### 3.3. 테스트 업데이트
- **[test_main.cpp](../test/test_native/test_main.cpp)**:
  - `test_night_sleep_mode` 테스트를 적응형 폴링 동작에 맞게 업데이트:
    - 200ms(250ms 미만) 경과 시 센서 값이 갱신되지 않음을 검증 (999.0f 유지)
    - 250ms 주기로 충분히 폴링한 후 중간값 필터가 정상 수렴함을 검증 (30.0f 도달)
    - 텔레메트리 비활성화는 기존과 동일하게 검증

## 4. 전력 절감 효과 분석

| 항목 | 주간 모드 | 야간 모드 (변경 전) | 야간 모드 (변경 후) |
|------|-----------|-------------------|-------------------|
| CPU 클럭 | 160 MHz | 160 MHz | **80 MHz** |
| Wi-Fi | ON | OFF | OFF |
| 센서 폴링 | 50ms (20회/초) | 차단 (0회/초) | **250ms (4회/초)** |
| 뚜껑 동작 | 정상 | **비활성** | **정상** |

> **핵심 개선:** 야간에도 뚜껑 개폐 기능을 유지하면서 CPU 클럭 50% 절감 + 센서 폴링 80% 절감을 달성했습니다.
