## 1. 이슈 개요 (Issue Overview)
기존 SmartBox의 CI/CD 및 모터 제어 동작에서 다음과 같은 세 가지 보완 사항이 식별되어 수정이 진행되었습니다.
1. **버전 고정 이슈:** GitHub Actions를 통한 CI 빌드 시 바이너리와 메타데이터(`version.json`)의 펌웨어 버전 정보가 정적 코드로 고정(예: `0.0.1`)되어 배포 후 실제 어떤 형상의 바이너리가 올라갔는지 추적이 어려운 문제가 존재했습니다.
2. **CI 비밀번호 파일 누락 이슈:** 웹 보안 기능(HTTP Basic Authentication) 도입 이후 로컬에서는 `secrets.h`에 자격 증명이 잘 관리되고 있었으나, CI 배포 파이프라인에서 동적으로 `secrets.h`를 빌드 환경에 구성할 때 `SECRET_DASHBOARD_USER` 및 `SECRET_DASHBOARD_PASS`가 누락되어 릴리즈 패키지 생성 시 로그인 관련 인증 헤더가 정의되지 않는 문제가 있었습니다.
3. **모터 전류 제한 임시 비활성화 요구:** 현재 뚜껑이 개폐될 때 모터 전류를 감시하여 과도한 부하 전류가 감지되면 비상 정지 상태(`STATE_EMERGENCY_STOP`)로 이행하는 하드웨어 보호 로직이 있으나, 하드웨어 테스트 및 디버깅을 위해 해당 과전류 감지 시 비상 모드로 강제 전환하는 부분을 임시로 비활성화(Disable) 처리할 필요가 있었습니다.

## 2. 해결 설계 및 구현 (Solution Design & Implementation)

### 2.1. Git Commit Hash 결합 동적 버저닝 구현
- **코드 수정:** [SmartBoxController.h](../src/SmartBoxController.h)에 프리프로세서 매크로 조건문을 사용해 빌드 타임에 외부에서 지정한 `FIRMWARE_VERSION_OVERRIDE`가 있다면 이를 버전 상수로 바인딩하고, 그렇지 않은 경우 기본 버전 `"0.0.1"`을 쓰도록 구현했습니다.
  ```cpp
  #ifdef FIRMWARE_VERSION_OVERRIDE
    static constexpr const char *FIRMWARE_VERSION = FIRMWARE_VERSION_OVERRIDE;
  #else
    static constexpr const char *FIRMWARE_VERSION = "0.0.1";
  #endif
  ```
- **CI 워크플로우 갱신:** [.github/workflows/deploy.yml](../.github/workflows/deploy.yml)에서 코드의 기본 버전을 `grep`하여 파싱한 뒤, 현재 빌드 대상의 Git short commit hash를 결합하여 동적으로 버전 문자열(예: `0.0.1-gab789b1`)을 구성합니다.
- 구성된 버전 문자열은 PlatformIO 빌드 시 환경변수 `PLATFORMIO_BUILD_FLAGS`를 통해 컴파일러 파라미터 `-DFIRMWARE_VERSION_OVERRIDE`로 주입되며, 배포 메타데이터인 `version.json` 정보 구성 시에도 일관성 있게 전달됩니다.

### 2.2. CI/CD `secrets.h` 대시보드 로그인 계정 누락 수정
- **CI 워크플로우 갱신:** [.github/workflows/deploy.yml](../.github/workflows/deploy.yml)의 `Create secrets.h from GitHub Secrets` 스크립트에 관리자 대시보드 인증 자격 증명 관련 매크로 생성을 명시적으로 추가했습니다.
  ```yaml
  #define SECRET_DASHBOARD_USER "${{ secrets.DASHBOARD_USER }}"
  #define SECRET_DASHBOARD_PASS "${{ secrets.DASHBOARD_PASS }}"
  ```
- 이 변경을 통해 배포 환경에서 동적 생성되는 헤더가 누락 없이 웹 보안 자격 증명값을 포함하여 최종 컴파일되도록 개선했습니다.

### 2.3. 모터 부하 과전류 비상 정지 임시 해제
- **과전류 예외 처리:** [SmartBoxController.cpp](../src/SmartBoxController.cpp)의 `STATE_OPENING` 및 `STATE_CLOSING` 제어 상태 내부에서 3회 연속 기준전류(`config.currentStallLimit`)를 초과하더라도 `forceAllRelaysOff()`와 `STATE_EMERGENCY_STOP`으로의 상태 전이를 임시 주석 처리했습니다.
- 과전류 감지 시 터미널 로그에는 `[WARN] MOTOR STALL DETECTED... (STALL PROTECTION DISABLED)` 경고를 지속해서 출력해 개발자가 디바이스 부하 상태를 시리얼 터미널을 통해 계속 모니터링할 수 있는 편의성은 유지했습니다.
- **TDD 단위 테스트 보완:** [test_main.cpp](../test/test_native/test_main.cpp)의 `test_stall_current_detection` 단위 테스트 코드도 임시 해제 스펙에 맞춰 수정되었습니다. 과전류 임계값 3회 이상 초과 시에도 비상 정지 상태로 전이되지 않고 계속 작동 중인 상태(`STATE_OPENING`)로 남아있음을 검증하도록 단언(Assert)을 재조정했습니다.
