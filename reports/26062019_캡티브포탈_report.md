# 📋 26062019_캡티브포탈_report.md

- **프로젝트명:** SmartBox
- **작성일자:** 2026년 06월 20일
- **주제:** 와이파이 프로비저닝 편의성 향상을 위한 Captive Portal (강제 포털) 도입

---

## 1. 이슈 개요 (Issue Overview)
SmartBox는 기존 Wi-Fi 연결에 실패하거나 새로운 환경에 설치될 시 자체 AP 모드(`SmartBox-WiFi`)를 켜고 접속을 대기합니다. 하지만 일반 사용자의 입장에서 무선랜 연결 후 웹 브라우저를 열고 수동으로 설정 IP 주소(`192.168.4.1`)를 입력하여 접속하는 일은 번거롭고 직관적이지 못합니다.
이 문제를 해결하기 위해, 사용자가 스마트폰이나 PC로 `SmartBox-WiFi` AP에 접속하는 즉시 Wi-Fi 설정 페이지가 자동으로 브라우저 팝업 화면으로 노출되도록 하는 **Captive Portal(강제 포털) 기능**의 도입이 필요하게 되었습니다.

## 2. 원인 분석 및 설계 (Root Cause & Design)
스마트 디바이스(iOS, Android, Windows 등)는 와이파이에 연결될 때 인터넷 사용 가능 여부를 검증하기 위해 백그라운드로 전용 도메인에 연결 테스트용 HTTP 요청(예: `http://captive.apple.com/hotspot-detect.html`, `/generate_204` 등)을 시도합니다.
이 과정에서 기기가 인터넷에 연결되어 있지 않으면 무선랜 설정 창이나 시스템 팝업을 통해 로그인 포털 페이지를 띄우게 됩니다.

따라서 캡티브 포털을 작동시키려면 다음 두 가지 논리적 요소가 필수적입니다:
1. **DNS 하이재킹 (DNS Hijacking):** 모든 인터넷 도메인 주소(예: `captive.apple.com`)에 대한 DNS 질의 요청을 강제로 SmartBox의 SoftAP 로컬 IP 주소(`192.168.4.1`)로 변환하여 응답해야 합니다.
2. **HTTP 리다이렉션 (HTTP Redirect):** 클라이언트가 DNS 확인을 마친 후 해당 도메인 경로로 HTTP 요청을 보내오면, 기기 내부 웹 서버가 이를 가로채 `http://192.168.4.1/` 홈 주소로 **HTTP 302 Found** 리다이렉트 응답을 반환해야 합니다. 단, 무한 루프 방지를 위해 이미 Host가 SoftAP IP인 경우의 404 예외 처리가 필요합니다.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. DNSServer 통합을 통한 DNS 하이재킹
- [WifiManager.h](../src/WifiManager.h) 및 [WifiManager.cpp](../src/WifiManager.cpp)에 ESP32 기본 탑재 라이브러리인 `<DNSServer.h>`를 통합했습니다.
- `WifiManager::dnsServer` 인스턴스를 static 멤버 변수로 정의하고 관리합니다.
- `WifiManager::init(...)` 실행 시 AP 모드가 구성된 후 `dnsServer.start(53, "*", WiFi.softAPIP())`를 호출하여 모든 호스트(`*`)에 대해 SoftAP IP 주소로 강제 응답하도록 포트 53번 DNS 서버를 시작했습니다.
- `WifiManager::update()`에 non-blocking 처리기인 `dnsServer.processNextRequest()`를 배치하여 끊김 없이 DNS 처리가 실행되도록 연결했습니다.
- Wi-Fi 비활성화(`stopWiFi()`) 및 STA 전용 모드 기동(`startWiFi(...)`) 시에는 불필요한 리소스 소비 및 포트 충돌 방지를 위해 `dnsServer.stop()`이 실행되도록 방어 로직을 구성했습니다.

### 3.2. Web Dashboard 리다이렉트 라우팅
- [WebDashboard.cpp](../src/WebDashboard.cpp) 내부 `init(...)` 함수에 `server.onNotFound()` 핸들러를 등록했습니다.
- 요청된 HTTP `Host` 헤더(`request->host()`)가 SoftAP IP 주소(`WiFi.softAPIP().toString()`)와 동일한지 비교 검증합니다.
- 호스트가 다를 경우(예: 모바일 기기의 자체 백그라운드 확인 도메인), SmartBox 설정 메인 루트인 `http://192.168.4.1/`로 `request->redirect()` (302 Redirect) 처리하여 로그인 포털 팝업을 유도합니다.
- 호스트가 이미 기기 IP 주소인 경우(정상 진입 상태에서 존재하지 않는 리소스를 호출한 상황 등), 무한 리다이렉션 루프에 빠지지 않도록 정상적인 **HTTP 404 Not Found** 에러를 응답합니다.
