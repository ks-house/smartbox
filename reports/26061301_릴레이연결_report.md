## 1. 이슈 개요 (Issue Overview)

ESP32-C6 제어 보드에 1채널 릴레이(메인 12V 전원 차단용)를 연결하여 구동 테스트를 진행하던 중, **시스템 부팅 후 1채널 릴레이의 상태 표시 LED(초록불)가 상시 작동(ON) 상태로 유지**되는 현상이 발생하였습니다.

* **현상**: 제어 핀(`GPIO 6`) 출력을 `LOW`와 `HIGH`로 변경하여 펌웨어를 업로드하여도 릴레이가 꺼지지 않고 동일하게 초록불이 계속 켜져 있음.
* **영향**: 액추에이터 대기 전력 차단 및 모터 극성 전환 시 안전 인터락(Interlock) 장치로서의 메인 차단기 역할이 동작하지 않아 쇼트(단락) 발생 위험이 존재함.

---

## 2. 기술적 원인 분석 (Root Cause Analysis)

본 이슈는 **3.3V Logic Level을 지원하는 ESP32 MCU**와 **5V 전원으로 구동되는 Active-Low(로우 레벨 트리거) 방식의 릴레이 모듈** 간의 하드웨어 전압 호환성 차이로 인해 발생하였습니다.

```
 [ 5V Active-Low Relay Module Input Stage Schematic ]

  VCC (+5V) ───────[ Resistor (e.g. 1kΩ) ]───────[ Optocoupler LED ]─────── Input Pin (IN)
                                                                                  │
                                                                                  ▼
                                                                           ESP32 GPIO 6
```

1. **Active-Low 제어 메커니즘**:
   - 해당 릴레이 모듈은 입력(IN) 핀이 `LOW`(0V)로 떨어져 다이오드에 전류가 흐를 때 작동(ON)합니다.
   - 반대로 작동을 끄기(OFF) 위해서는 입력(IN) 핀의 전압이 전원 전압(`VCC` = 5V)에 수렴하여 다이오드 양단에 전류가 흐르지 않아야 합니다.
2. **논리 레벨 전압 차이**:
   - ESP32-C6의 GPIO 핀은 `HIGH` 상태일 때 **3.3V**를 출력합니다.
   - 이때 릴레이 입력단 회로의 내부 전압 차는 $5\text{V} - 3.3\text{V} = 1.7\text{V}$가 됩니다.
   - 일반적으로 릴레이 모듈 내부 포토커플러(Optocoupler) 다이오드의 순방향 전압 강하($V_f$)는 **1.2V ~ 1.4V** 수준입니다.
   - 따라서 ESP32가 `HIGH`를 출력하더라도 전압 차가 다이오드의 문턱 전압보다 크기 때문에 미세 전류가 흐르게 되고, 결과적으로 포토커플러가 활성화되어 **릴레이가 상시 작동(ON) 상태를 유지**합니다.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

하드웨어 추가 결선이나 부품 추가가 불가능한 상황에서, 제어 핀의 **High-Impedance(고임피던스, floating) 특성**을 활용하는 소프트웨어 우회 방법(Option 3)을 적용하였습니다.

### 3.1. 제어 논리 동작 설계

* **릴레이 OFF (대기 상태)**:
  - 제어 핀을 `INPUT` 모드로 설정합니다.
  - 이 상태에서는 핀이 고임피던스 상태가 되어 내부로 전류가 흐를 수 없습니다.
  - 릴레이 모듈 내부의 자체 풀업 저항에 의해 `IN` 핀 전압이 완전히 **5V**에 수렴하여 포토커플러가 확실하게 꺼집니다.
* **릴레이 ON (동작 상태)**:
  - 제어 핀을 `OUTPUT` 모드로 변경한 뒤 `LOW`(0V) 신호를 줍니다.
  - 핀이 GND로 연결(Pull-down)되면서 전류가 정상적으로 흘러 릴레이가 안정적으로 켜집니다.

### 3.2. 소스 코드 수정 (main.cpp)

[main.cpp](../src/main.cpp)에 다음과 같이 제어 코드를 수정하였습니다.

```cpp
// 1. setup() 함수 내 초기화 변경
void setup() {
  Serial.begin(115200);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);

  // 1채널 릴레이 초기 상태: OFF (INPUT 모드로 설정하여 전류 흐름 차단)
  pinMode(RELAY_MAIN_PIN, INPUT);
  
  pinMode(RELAY_DIR_A_PIN, OUTPUT);
  pinMode(RELAY_DIR_B_PIN, OUTPUT);

  // 부팅 시 모든 릴레이 전원 강제 차단 (2채널 방향성 초기화)
  digitalWrite(RELAY_DIR_A_PIN, HIGH);
  digitalWrite(RELAY_DIR_B_PIN, HIGH);
}

// 2. loop() 함수 내 개폐 시퀀스 제어 변경
void loop() {
  long dist = getDistance();
  if (dist > 0 && dist < DIST_THRESHOLD) {
    
    // [STEP 1] 1채널 메인 전원 ON (OUTPUT 및 LOW)
    pinMode(RELAY_MAIN_PIN, OUTPUT);
    digitalWrite(RELAY_MAIN_PIN, LOW);
    delay(100);

    // [STEP 2] 2채널 릴레이 정회전 (뚜껑 열기)
    digitalWrite(RELAY_DIR_A_PIN, LOW);
    digitalWrite(RELAY_DIR_B_PIN, HIGH);
    delay(ACTUATOR_TIME);

    // [STEP 3] 대기 시간 동안 메인 전원 OFF (INPUT 전환하여 절전 및 안전 확보)
    digitalWrite(RELAY_DIR_A_PIN, HIGH);
    digitalWrite(RELAY_DIR_B_PIN, HIGH);
    pinMode(RELAY_MAIN_PIN, INPUT); 
    delay(WAIT_TIME);

    // [STEP 4] 1채널 메인 전원 다시 ON (OUTPUT 및 LOW)
    pinMode(RELAY_MAIN_PIN, OUTPUT);
    digitalWrite(RELAY_MAIN_PIN, LOW);
    delay(100);

    // [STEP 2-2] 2채널 릴레이 역회전 (뚜껑 닫기)
    digitalWrite(RELAY_DIR_A_PIN, HIGH);
    digitalWrite(RELAY_DIR_B_PIN, LOW);
    delay(ACTUATOR_TIME);

    // [STEP 5] 완료 및 대기 모드 (INPUT 전환하여 완전 차단)
    digitalWrite(RELAY_DIR_A_PIN, HIGH);
    digitalWrite(RELAY_DIR_B_PIN, HIGH);
    pinMode(RELAY_MAIN_PIN, INPUT);
    delay(3000);
  }
  delay(500);
}
```

## 4. 검증 결과 (Verification Results)

수정된 펌웨어를 컴파일하여 ESP32-C6 디바이스에 업로드하고 기능 검증을 실시하였습니다.

1. **초기 부팅 테스트**:
   - 전원 인가 시 1채널 릴레이의 신호 LED(초록색)가 꺼진 채로 유지됨을 확인하였습니다. (정상 작동)
2. **동작 시퀀스 테스트**:
   - 초음파 센서 감지 거리 내 물체 접근 시 1채널 메인 릴레이에 초록불이 켜지며 12V 전원이 투입됨을 확인하였습니다.
   - 열림 완료 후, 5초 대기 시간 동안 초록불이 확실하게 꺼지며 대기 전력이 완전 차단됨을 확인하였습니다.
   - 닫힘 시퀀스 시작 시 다시 초록불이 들어오며 동작 후 최종적으로 복귀하여 꺼졌습니다.
3. **2채널 연동 테스트**:
   - 2채널 릴레이 방향 제어선과 1채널 메인 전원선 간의 기계적·시간차 인터락이 안전하게 연동되며 쇼트 현상 없이 정상 제어됨을 확인 완료하였습니다.

## 5. 결론 및 향후 권장 사항 (Conclusion & Recommendations)

* **결론**: 소프트웨어 핀 모드 동적 전환 기술을 적용하여 추가 하드웨어 레벨 시프팅 부품 없이 3.3V 단독 제어로 5V Active-Low 릴레이의 상시 오작동 문제를 완벽히 해결하였습니다.
* **권장 사항**:
  - 현재 꼼수 제어는 안정적으로 작동하나, 추후 장기 운용을 하거나 다른 고정형 모듈을 확장할 경우, 회로의 신뢰성 및 잡음 내성 향상을 위해 **3.3V-5V 양방향 논리 레벨 시프터(Logic Level Shifter)** 또는 **NPN 트랜지스터(BC547 등)**를 추가하는 정석적인 하드웨어 설계를 반영할 것을 권장합니다.
