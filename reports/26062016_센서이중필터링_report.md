- **주제:** 센서이중필터링 (초음파 센서 이중 필터링 및 디바운스 구현)
    
---
    
## 1. 이슈 개요 (Issue Overview)
야외에 설치되는 초음파 거리 센서(AJ-SR04T)의 Raw 데이터(`hw.getDistanceCm()`)를 그대로 사용할 경우, 비, 벌레, 미세먼지 및 다양한 하드웨어적 전기 노이즈에 의해 일시적으로 거리 값이 튀는 현상(Spike)이 발생합니다. 이로 인해 사람이 접근하지 않았음에도 뚜껑이 열리는 **'Ghost Opening'** 현상이 초래될 위험이 있었습니다.

## 2. 원인 분석 (Root Cause Analysis)
- **센서 Raw 데이터 튀임 (Sensor Spike):** 초음파 센서의 특성상 단일 측정이 극도로 짧은 시간에 튈 수 있어 단일 임계치 검사만으로는 오작동을 필터링할 수 없음.
- **연속 검증 결여:** 접근 신호 발생 시 사람이 실제로 특정 시간 동안 유지하고 있는지 시간 기반으로 확인하는 시간 디바운싱(Time-based Debouncing) 로직이 없어 순간적으로 스쳐 지나가는 낙엽이나 이물질에도 센서가 반응하여 오작동이 유발됨.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. 튜닝 상수 정의
사용자가 손쉽게 상수를 튜닝할 수 있도록 클래스 상단 및 설정 헤더에 다음과 같이 상수를 명확히 정의하였습니다.
```cpp
// Tuning Constants
static constexpr float OPEN_THRESHOLD_CM = 50.0f;
static constexpr unsigned long DEBOUNCE_DELAY_MS = 300;
```

### 3.2. 1단계: 중간값 필터 (Median Filter) 구현
최근 5개의 거리 데이터를 링 버퍼에 저장하고 버블 정렬을 통해 비정상적으로 튄 최대/최소값을 배제한 순수 중간값을 추출하는 1차 필터링을 구축하였습니다.
- **링 버퍼 멤버 추가:** `float distanceHistory[5]`, `int distanceHistoryIdx`
- **중간값 추출 메서드:** `getFilteredDistance()`
- 센서 측정 시마다 `updateDistanceBuffer()`를 통해 링 버퍼에 신규 센서값을 갱신하고 `getFilteredDistance()`를 호출하여 `currentDistance`를 업데이트합니다.

### 3.3. 2단계: 시간 기반 디바운싱 (Time-based Debouncing) 구현
추출된 필터링 거리 데이터가 임계값 이내로 들어올 때 즉시 상태를 천이하지 않고 300ms 이상 유지되는 경우에만 문을 열도록 설계하였습니다.
- **시간 트래킹 멤버 추가:** `unsigned long lastDetectStartTime`
- **IDLE 상태 천이 로직:**
  - `currentDistance`가 `config.distThreshold` (기본값 `OPEN_THRESHOLD_CM`) 이내로 처음 검출 시 `lastDetectStartTime = hw.getMillis()`로 타이머를 시작.
  - 해당 상태가 연속으로 `config.debounceDelay` (기본값 `DEBOUNCE_DELAY_MS` = 300ms) 이상 유지되는 순간 `STATE_OPEN_START`로 천이.
  - 도중에 센서 거리가 임계값을 벗어나면 `lastDetectStartTime = 0`으로 즉시 초기화하여 일시적인 간섭을 완전히 배제.
- **전이 시 리셋 방어 로직:** `transitionTo()` 호출 시 항상 `lastDetectStartTime = 0`으로 리셋하여 잔류 시간 노이즈가 다른 상태 천이에 개입하지 않도록 방어.

### 3.4. 절전 및 예외 상태 관리
- `nightSleepActive`가 참이거나 `STATE_OTA_UPDATING` 상태인 경우 센서의 주기적 폴링(50ms 주기)을 완전히 억제하여 불필요한 전류 소모 및 동작 간섭을 예방하였습니다.

## 4. 검증 결과 (Verification Results)

### 4.1. 유닛 테스트(`test_main.cpp`) 보완 및 검증
- **디바운스 시뮬레이션 적용:** 기존의 5회 업데이트 시뮬레이션을 15회(총 750ms)로 확장하여 링 버퍼 충진 및 300ms 디바운스 타이머 검증을 충족시켰습니다.
- **`test_ultrasonic_median_filter` 개선:** `Noise Test 2`에서 노이즈 입력 5회 수행 직후(시간상 50ms 경과)에는 여전히 `STATE_IDLE` 상태를 안정적으로 유지하다가, 추가적으로 6회(총 350ms 연속 검출) 지속 검출되는 시점에 비로소 `STATE_OPEN_START`로 전환되는 것을 검증하였습니다.
- **`test_night_sleep_mode` 개선:** 야간 절전 모드 진입 시 거리 센서 접근 자극(30cm)을 15회 주입하더라도 센서 폴링이 억제되어 문이 열리지 않고 안정적으로 `STATE_IDLE`을 유지함을 증명하였습니다.
