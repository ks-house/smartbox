## 1. 이슈 개요 및 배경 (Overview)

스마트 자동 수거함(SmartBox) 프로젝트의 하드웨어 안전(Safety-First)을 보장하고, 테스트 주도 개발(TDD) 방법론을 활용하여 고품질의 소프트웨어 아키텍처를 설계·구현하고 실제 ESP32-C6 타겟 보드 배포까지 완료하였습니다. 

이후 물리 검증 단계에서 발생한 **"뚜껑이 열린 후 10초간 대기하지 않고, 사용자가 손을 떼자마자 즉시 닫혀버리는 현상"**을 해결하기 위해 대기 상태(`STATE_HOLD`) FSM 로직을 최적화하고, 안전한 타이머 리셋 매커니즘을 추가 수립해 최종 검증을 완수했습니다.

## 2. 주요 아키텍처 및 단계별 구현 내역 (Core Architecture)

### 2.1. 하드웨어 추상화 레이어 (HAL) 도입
* **추상 인터페이스 구현:** [HardwareInterface.h](../src/HardwareInterface.h)를 수립하여 메인 FSM 제어 로직과 물리적인 GPIO 및 센서 결선(I2C, pulseIn 등)을 완전히 분리했습니다.
* **이중 구현체 분리:** 실제 ESP32 아두이노 Core API를 호출하는 [Esp32Hardware.cpp](../src/Esp32Hardware.cpp)와 호스트 PC 환경 및 타겟 시뮬레이션 테스트를 지원하는 [MockHardware.h](../src/MockHardware.h)를 작성하여 코드 변경 없이 즉각적인 단위 테스트 실행이 가능하도록 했습니다.

### 2.2. 영구 설정 보존 및 자가 치유 가드
* [ConfigManager.cpp](../src/ConfigManager.cpp)를 통해 ESP32 내장 비휘발성 플래시 영역(Preferences)에 센서 감도, 대기 시간, 모터 과전류 스톨 감도 등을 영구 보존합니다.
* 부팅 시 NVS 공간 손상 등으로 인해 설정값이 `0`이나 비정상적인 값으로 읽힐 경우, 사전에 정의된 안전 디폴트 값으로 강제 자동 리셋하는 **자가 치유(Self-healing) 로직**을 가동해 오작동을 원천 봉쇄했습니다.

### 2.3. 비동기 웹 대시보드 및 IoT 연동
* [WebDashboard.cpp](../src/WebDashboard.cpp): `ESPAsyncWebServer` 기반의 고급 다크 모드 대시보드를 구축해 실시간 텔레메트리 스트리밍(거리, 전압, 전류 등), 동적 설정 변경, 수동 개방 및 비상 정지 해제를 제공합니다.
* [SmartThingsReporter.cpp](../src/SmartThingsReporter.cpp): FreeRTOS 백그라운드 큐와 태스크 분리 설계를 채택하여, Wi-Fi 네트워크 단절이나 웹훅 전송 지연이 발생하더라도 모터 차단 및 센서 감지와 같은 안전 제어 루프를 블로킹하지 않도록 하였습니다.

## 3. HOLD 상태 10초 대기 및 리셋 최적화 (Wait Optimization)

사용자가 뚜껑 개방 후 쓰레기를 편안하게 버릴 수 있도록 대기 상태 FSM 로직을 다음과 같이 수정 보완했습니다.

* **조기 폐쇄(Early Close) 제거:** 
  기존에는 초음파 센서 감지 거리 밖으로 물체가 벗어날 때 즉시 닫히게 하는 조기 폐쇄 조건(`currentDistance > config.distThreshold && currentDistance < 990.0f`)이 가동되어, 사용자가 손을 떼자마자 문이 닫혔습니다. 이 조건을 완전히 제거해 사용자가 물러나더라도 최소 설정된 시간(기본 10초, 10000ms) 동안 문이 열려있도록 고정했습니다.
* **근접 재감지 타이머 리셋:**
  대기 10초 카운트다운이 흘러가는 도중이라도 사용자가 다시 센서 감지 범위(cm) 이내로 접근하면, 대기 타이머를 즉시 현재 시간으로 갱신하여 사용자가 센서 앞에 머무는 동안은 뚜껑이 계속 열려 있도록 개선하였습니다.

## 4. TDD 및 타겟 유닛 테스트 검증 결과 (Verification)

실제 ESP32-C6 보드에 테스트 펌웨어를 업로드하여 실행하는 `esp32-c6-test` 환경에서 Unity 프레임워크를 기반으로 유닛 테스트를 진행했습니다. 신규 대기 시간 패치에 대한 테스트를 추가한 결과, **총 7가지의 핵심 유닛 테스트 시나리오가 모두 통과(PASSED)**되었습니다.

## 5. 결론 및 향후 권장 사항 (Conclusion & Recommendations)

* **결론:** 소프트웨어 FSM 튜닝을 통해 물리 결선 수정 없이 "뚜껑 열림 10초 대기 및 리셋" 동작을 성공적으로 이끌어냈습니다. 모든 구현은 Unity 단위 테스트를 통해 안전성이 정량적으로 보장되었으며, 실제 ESP32-C6에 펌웨어 업로드 및 사용자 매뉴얼 문서화까지 완벽히 마쳤습니다.
* **향후 권장 사항:**
  1. 현재는 Wi-Fi 개방형 AP(`SmartBox-WiFi`) 상태로 통신 중입니다. 실제 실외나 다수가 사용하는 지역에 설치할 경우 웹 대시보드 보안 향상을 위해 WPA2 비밀번호를 강제 적용하는 패치를 권장합니다.
  2. 초음파 센서 렌즈 표면에 이물질이나 빗물 등이 묻어 튀는 현상이 심해질 경우 대기 타이머가 의도치 않게 계속 리셋될 우려가 있습니다. 대시보드를 통해 현장 환경에 맞춰 감지 거리 임계값(`distThreshold`)을 주기적으로 캘리브레이션할 것을 권장합니다.
