# 📋 26062018_수동청소모드_report.md

- **프로젝트명:** SmartBox
- **작성일자:** 2026년 06월 20일
- **주제:** 수동 청소 모드 (Hold-Open Mode) 지원 및 웹 대시보드 연동

---

## 1. 이슈 개요 (Issue Overview)
작업자가 쓰레기 봉투를 교체하거나 SmartBox 내부를 청소할 때, 초음파 센서 감지로 인해 뚜껑이 계속 닫히는 불편함이 존재했습니다. 안전하고 편리한 유지보수 작업을 위해 웹 대시보드에서 원격으로 제어 가능한 **수동 청소 모드(Maintenance/Hold-Open Mode)**의 구현이 요구되었습니다.

요구 사양:
- 수동 청소 모드 진입 시 즉시 뚜껑을 개방하고 5분(300,000ms) 동안 열림 상태를 강제 유지.
- 해당 상태에서는 초음파 센서의 거리 변화(사람의 접근 또는 멀어짐 등)를 완전히 무시.
- 대기 시 릴레이 제어 전류를 완벽 차단(0mA 대기전력 상태)하여 안전 및 에너지 절약 도모.
- 5분 시간 초과 시 자동으로 뚜껑을 닫고 일반 대기(`STATE_IDLE`) 상태로 자동 복귀.
- 사용자가 웹 대시보드에서 즉시 종료 요청을 보낼 시 즉각 뚜껑을 닫음.

## 2. 원인 분석 및 설계 (Root Cause & Design)
기존 FSM(유한 상태 머신)은 센서 감지에 따른 자동 개폐 흐름(`STATE_IDLE` -> `STATE_OPENING` -> `STATE_HOLD` -> `STATE_CLOSING`)만을 지원했습니다. 이 흐름을 방해하지 않으면서도 안정적으로 수동 제어가 개입할 수 있도록 FSM 구조를 확장하고, 멀티스레딩 환경의 데이터 무결성을 위해 `std::recursive_mutex`를 이용한 스레드 세이프 설계를 적용했습니다.

### 상태 천이도 및 안전 설계
- **`STATE_MAINTENANCE` 상태 도입:** 뚜껑이 열려 대기하는 상태를 기존 `STATE_HOLD`(10초 대기 후 자동 닫힘)와 구분하여, 5분 타이머로 별도 관리합니다.
- **안전 연동(Relay Isolation):** `STATE_MAINTENANCE` 진입 완료 시, 릴레이의 신호선과 주전원을 차단하는 `setRelayStates(false, false, false)`를 명시적으로 수행하여 오작동 방지 및 대기 전류를 차단합니다.
- **비동기 스레드 안전:** 웹 서버의 비동기 핸들러가 `SmartBoxController` 내부 필드 및 상태를 수정할 때 데드락이나 데이터 레이스가 발생하지 않도록 `std::recursive_mutex` 멤버 변수인 `dataMutex`로 FSM 변경 작업을 보호합니다.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. FSM 상태 머신 확장
- [SmartBoxController.h](../src/SmartBoxController.h)에 `STATE_MAINTENANCE` 상태 및 5분 타임아웃(`MAINTENANCE_TIMEOUT_MS = 300000`) 상수 정의.
- [SmartBoxController.cpp](../src/SmartBoxController.cpp)의 `update()` 함수에 `STATE_MAINTENANCE` 상태 제어 로직을 추가하여 타임아웃 경과 시 `STATE_CLOSE_START`로 천이하도록 구현.
- `startMaintenanceMode()`, `stopMaintenanceMode()` 및 `getMaintenanceRemainingSeconds()` 메서드를 구현하여 외부 API 호출과 연동.

### 3.2. 웹 대시보드 API 신설 및 UI 개편
- [WebDashboard.cpp](../src/WebDashboard.cpp)에 `/api/maintenance` (GET/POST) 비동기 API 엔드포인트를 등록하여 쿼리 파라미터 `action=start` 및 `action=stop`에 따라 수동 청소 모드를 기동/중지하도록 처리.
- `/api/status` API 응답 JSON에 `"maintenance": true/false` 및 `"maintenanceTime": 남은시간(초)`을 추가하여 실시간 동기화.
- UI 대시보드 화면 하단에 **[🧹 수동 청소 모드 (5분 열림)]** 버튼을 추가하고, 활성화 여부에 따라 버튼 색상 변경(주황색 -> 빨간색) 및 남은 시간 카운트다운 텍스트를 실시간으로 업데이트.

### 3.3. 텔레메트리 연동
- [TelemetryManager.cpp](../src/TelemetryManager.cpp) 내 `stateToString()` 헬퍼에 `STATE_MAINTENANCE`를 추가하여 InfluxDB에 장치의 현재 상태가 `"MAINTENANCE"`로 안전하게 전송되도록 매핑.

4. 300,000ms 경과 시 자동으로 `STATE_CLOSE_START` -> `STATE_CLOSING`으로 복귀하는지 검증.
