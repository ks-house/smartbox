## 1. 이슈 개요 (Issue Overview)

코드 리뷰를 통해 3가지 치명적 결함이 발견되었습니다.

| # | 결함 | 분류 |
|---|------|------|
| 1 | `STATE_OPENING` / `STATE_CLOSING` 과전류 비상정지 로직이 주석 처리(`/* */`)로 완전 무력화 | 🚨 하드웨어 안전 |
| 2 | 야간 절전 해제 시 `startWiFi()`가 `WIFI_STA` 전용 모드로만 복귀 → Soft AP 소멸로 관리자 접속 영구 차단 | 🌙 가용성 |
| 3 | 재부팅 후 `STATE_STARTUP_OPEN` 진입 시 사람이 접근하지 않으면 무한히 열려있음 | ⏰ 가용성 |

## 2. 원인 분석 (Root Cause Analysis)

### 결함 1: 협착 방지 로직 무력화
`SmartBoxController.cpp`의 `STATE_OPENING` / `STATE_CLOSING` 상태 내부에서 과전류 감지 후 비상정지를 수행하는 핵심 코드가 `// Temporarily disabled as requested by the user` 주석과 함께 C 스타일 블록 주석(`/* */`)으로 감싸져 있었습니다. 결과적으로 모터에 무제한 전류가 공급되는 상태에서도 릴레이가 차단되지 않아 물리적 협착 사고 및 릴레이 소손 위험이 상존했습니다.

### 결함 2: 야간 절전 해제 시 AP 미복구
`WifiManager::startWiFi()`가 `WiFi.mode(WIFI_STA)` 전용 모드로만 설정하여 `init()` 시점에 개설된 Soft AP(`SmartBox-WiFi`)가 복구되지 않았습니다. 또한 `dnsServer.start()`도 재호출되지 않아 Captive Portal이 소멸했습니다. AP SSID/Password를 기억하는 정적 변수도 없어 복구 자체가 구조적으로 불가능한 상태였습니다.

### 결함 3: STATE_STARTUP_OPEN 무한 개방
`STATE_STARTUP_OPEN` 상태에는 사람 접근 감지 시 `STATE_HOLD`로 전이하는 로직만 있었고, 아무도 오지 않는 경우의 타임아웃 처리가 누락되어 있었습니다. 재부팅 후 뚜껑이 열린 상태이면 영원히 열린 채로 유지되었습니다.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. 협착 방지 비상정지 로직 복원

**[SmartBoxController.cpp](../src/SmartBoxController.cpp)**

`STATE_OPENING`과 `STATE_CLOSING` 양쪽에서 비활성화 주석을 제거하고 실제 코드를 활성화했습니다.

```cpp
// STATE_OPENING (개방 중) — 복원된 코드
if (openStallCount >= 3) {
    Serial.printf("[WARN] MOTOR STALL DETECTED DURING OPEN! ...\n");
    openStallCount = 0;
    forceAllRelaysOff();
    transitionTo(STATE_EMERGENCY_STOP);
    break;
}

// STATE_CLOSING (닫힘 중) — 복원된 코드
if (closeStallCount >= 3) {
    Serial.printf("[WARN] MOTOR STALL DETECTED DURING CLOSE! ...\n");
    closeStallCount = 0;
    forceAllRelaysOff();
    transitionTo(STATE_EMERGENCY_STOP);
    break;
}
```

**동작 시퀀스:**
- 500ms 인러시 전류 바이패스 이후 활성화
- `config.currentStallLimit`(기본 3000mA) 초과 시 카운터 증가
- 3회 연속 초과 시: `forceAllRelaysOff()` → 전체 릴레이 INPUT 모드(고임피던스) → `STATE_EMERGENCY_STOP` 전이
- `emergencyRecoveryTime`(기본 5초) 경과 후 자동 복구

**[test_main.cpp](../test/test_native/test_main.cpp)**

`test_stall_current_detection` Assertion을 안전 로직 복원에 맞게 수정했습니다.

-TEST_ASSERT_EQUAL(STATE_OPENING, controller.getCurrentState()); // 비활성 상태

### 3.2. 야간 절전 해제 시 AP & Captive Portal 복구

**[WifiManager.h](../src/WifiManager.h)**

AP 자격증명을 기억하기 위한 정적 멤버 추가:
```cpp
static String _apSsid;
static String _apPass;
```

**[WifiManager.cpp](../src/WifiManager.cpp)**

- `init()`: `_apSsid`와 `_apPass`에 AP 자격증명을 저장
- `startWiFi()`: `WIFI_STA` → **`WIFI_AP_STA`** 듀얼 모드로 변경, `WiFi.softAP()` 재호출, `dnsServer.start(53, "*", ...)` 재시작

```cpp
void WifiManager::startWiFi(const char* ssid, const char* password) {
    WiFi.mode(WIFI_AP_STA);  // STA 전용 → AP_STA 듀얼 모드
    if (_apSsid.length() > 0) {
        WiFi.softAP(_apSsid.c_str(), _apPass.c_str()); // AP 복구
    }
    dnsServer.start(53, "*", WiFi.softAPIP()); // Captive Portal 복구
    // ... STA 연결
}
```

**복구 흐름:**
1. 야간 절전 해제(`PowerManager::update()`)
2. `WifiManager::startWiFi(savedSsid, savedPass)` 호출
3. `WIFI_AP_STA` 모드 설정 → `SmartBox-WiFi` AP 자동 복구
4. Captive Portal DNS 재시작 → `http://192.168.4.1` 접속 정상화
5. 외부 공유기 STA 연결 재시도

### 3.3. STATE_STARTUP_OPEN 무한 개방 방지 (30초 타임아웃)

**[SmartBoxController.cpp](../src/SmartBoxController.cpp)**

```cpp
case STATE_STARTUP_OPEN:
    setRelayStates(false, false, false);
    if (currentDistance > 0 && currentDistance < config.distThreshold && !isCooldown) {
        transitionTo(STATE_HOLD);
    }
    // Timeout guard: auto-close after 30 seconds if no human approaches
    if (hw.getMillis() - stateTimer >= 30000) {
        Serial.println("[STATE] STATE_STARTUP_OPEN timeout (30s). Auto-closing.");
        transitionTo(STATE_CLOSE_START);
    }
    break;
```

- `stateTimer`는 `transitionTo()` 시점에 `hw.getMillis()`로 갱신됨
- 사람 접근 시 → `STATE_HOLD` (기존 동작 유지)
- 30초 무접근 시 → `STATE_CLOSE_START` → `STATE_CLOSING` 자동 실행
