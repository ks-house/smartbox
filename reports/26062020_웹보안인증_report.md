# 📋 26062020_웹보안인증_report.md

- **프로젝트명:** SmartBox
- **작성일자:** 2026년 06월 20일
- **주제:** Web Dashboard 및 API 보안을 위한 HTTP Basic Authentication 도입

---

## 1. 이슈 개요 (Issue Overview)
기존 SmartBox의 웹 대시보드 및 제어용 REST API는 비인가자 접근 제한 없이 노출되어 있었습니다. 로컬 네트워크 또는 SoftAP 모드에 접속한 모든 사용자가 인증 없이 쓰레기통 뚜껑을 열거나(`forceOpen`), 설정을 조작하고, Wi-Fi 자격 증명을 훔치거나 임의로 펌웨어 업데이트를 수행할 수 있는 심각한 보안 취약점이 존재했습니다.
이를 해결하고 상용 IoT 수준의 장치 보안을 확보하기 위해, 중요 엔드포인트에 대한 **HTTP Basic Authentication(기본 인증)**을 도입해야 합니다.

## 2. 원인 분석 및 설계 (Root Cause & Design)
비동기 웹 서버인 `ESPAsyncWebServer`는 비인가 요청 발생 시 클라이언트에 브라우저 표준 로그인 팝업을 띄울 수 있는 `request->requestAuthentication()` 메서드와 매개변수로 사용자명 및 패스워드를 받아 대조하는 `request->authenticate(user, pass)` 기능을 자체 내장하고 있습니다.

보안 설계 원칙:
1. **중요 엔드포인트 보안 적용:** 대시보드 화면(`/`) 및 제어/조회 API 엔드포인트(`/api/...`) 전반에 인증을 필수 적용합니다.
2. **보안 민감 정보(Secrets) 분리:** 계정 정보는 코드에 하드코딩하지 않고 매크로 형태로 `include/secrets.h`에 정의하고, 배포용 템플릿인 `secrets.h.example`도 최신화하여 관리합니다.
3. **인터렉션 편의성 유지:** 브라우저 표준 Basic Auth의 특징을 살려 홈화면(`/`) 인증 성공 이후에는 브라우저가 모든 하부 API 통신 시 인증 헤더(`Authorization`)를 자동으로 주입하도록 연동하여, 추가 로그인 과정 없이 비동기 API 갱신(`setInterval` status 등)이 원활하게 연동됩니다.
4. **캡티브 포털 예외 처리:** 초기 연결 시 Captive Portal이 작동할 수 있도록 `onNotFound` wildcard 리다이렉트 핸들러는 인증 대상에서 제외합니다.

## 3. 해결 설계 및 구현 (Solution Design & Implementation)

### 3.1. 인증 자격 증명 설정 분리
- [secrets.h](../include/secrets.h) 파일에 대시보드 관리자 인증 계정 정보를 정의했습니다.
  ```cpp
  #define SECRET_DASHBOARD_USER "admin"
  #define SECRET_DASHBOARD_PASS "smartbox123!"
  ```
- 공용 저장소 배포용인 [secrets.h.example](../include/secrets.h.example) 파일에도 동일한 계정 정보 템플릿 상수를 추가하여 형상을 동기화했습니다.

### 3.2. Web Dashboard 인증 적용 및 헬퍼 구현
- [WebDashboard.cpp](../src/WebDashboard.cpp) 상단에 요청 헤더에서 계정 정보를 확인하는 `checkAuth(AsyncWebServerRequest *request)` 정적 헬퍼 함수를 추가했습니다.
  ```cpp
  static bool checkAuth(AsyncWebServerRequest *request) {
      if (!request->authenticate(SECRET_DASHBOARD_USER, SECRET_DASHBOARD_PASS)) {
          request->requestAuthentication();
          return false;
      }
      return true;
  }
  ```
- 등록된 라우팅 핸들러의 실행 진입 시점 상단에 `if (!checkAuth(request)) return;` 가드를 추가하여 비인가 요청 시 `401 Unauthorized` 표준 응답을 내려주고 실행을 즉시 인터럽트하도록 처리했습니다.
- 인증이 걸린 대상 라우트:
  - `/` (메인 대시보드 HTML)
  - `/api/status` (실시간 상태 정보 조회)
  - `/api/release` (비상 정지 해제)
  - `/api/open` (원격 강제 개방)
  - `/api/maintenance` (유지보수 모드 제어)
  - `/api/config` (센서 감도 및 오타 설정 변경)
  - `/api/wifi/scan` (주변 네트워크 AP 탐색)
  - `/api/wifi/connect` (네트워크 설정 프로비저닝)
  - `/api/update-from-nas` (HTTPS NAS OTA 강제 기동)
