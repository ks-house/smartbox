## 1. 이슈 개요 (Issue Overview)
기존에 구현된 무인 자동 업데이트(Auto-OTA) 시스템의 정기 실행 시점이 새벽 3시로 고정되어 있어 사용 환경이나 관리자의 스케줄 변경에 맞춰 유연하게 예약 시간을 변경할 수 없는 제약이 있었습니다. 이를 해결하기 위해 사용자가 내장 웹 대시보드를 통해 실행 예약 시간(0~23시)을 직접 지정 및 동적 캘리브레이션하고, 설정이 재부팅 후에도 유지되도록 NVS 영구 저장소에 연동하는 작업을 진행했습니다.

## 2. 해결 설계 및 구현 (Solution Design & Implementation)

### A. 설정 데이터 구조 확장 ([src/SmartBoxController.h](file:///c:/Users/shcat/Documents/PlatformIO/Projects/smartbox/src/SmartBoxController.h))
`BoxConfig` 구조체에 예약 시간을 저장할 `otaHour` 필드를 추가했습니다 (기본값: `3`).
```cpp
struct BoxConfig {
  // ... 기존 설정 속성들
  int otaHour = 3; // 예약 업데이트 실행 시간 (0 ~ 23)
};
```

### B. 영구 NVS Preferences 연동 ([src/ConfigManager.cpp](file:///c:/Users/shcat/Documents/PlatformIO/Projects/smartbox/src/ConfigManager.cpp))
- `loadConfig()` 시 Preferences 네임스페이스로부터 `otaHour`를 읽어들이고, 범위 확인 가드 로직을 거쳐 오동작(잘못 입력된 0 이하 또는 24 이상의 값)에 대해 자가 치유(Self-Healing) 하도록 설계했습니다.
- `saveConfig()` 시 Preferences 네임스페이스에 `otaHour` 값을 정수형(`putInt`)으로 안전하게 쓰도록 처리했습니다.
```cpp
// loadConfig
config.otaHour = prefs.getInt("ota_hour", config.otaHour);
if (config.otaHour < 0 || config.otaHour > 23) {
    config.otaHour = 3;
}
// saveConfig
prefs.putInt("ota_hour", config.otaHour);
```

### C. 비동기 스케줄러 동적 시간 비교 ([src/AutoOtaManager.cpp](file:///c:/Users/shcat/Documents/PlatformIO/Projects/smartbox/src/AutoOtaManager.cpp))
기존에 `timeinfo.tm_hour == 3`으로 고정되어 있던 스케줄링 대조 조건식을 컨트롤러의 캘리브레이션 값 `otaHour`를 읽어 동적으로 대조하도록 변경했습니다.
```cpp
    int scheduledHour = controllerPtr->getConfig().otaHour;
    if (timeinfo.tm_hour == scheduledHour) {
        if (lastOtaCheckDay != timeinfo.tm_mday) {
            lastOtaCheckDay = timeinfo.tm_mday;
            Serial.printf("[AUTO-OTA] Scheduled trigger time (%d:00 KST) reached. Launching AutoOtaTask...\n", scheduledHour);
            startOtaUpdate(false);
        }
    }
```

### D. 웹 대시보드 API 확장 및 UI 구현 ([src/WebDashboard.cpp](file:///c:/Users/shcat/Documents/PlatformIO/Projects/smartbox/src/WebDashboard.cpp))
1. **API 노출:** `/api/status`에서 반환되는 JSON 스트링 설정 항목에 `otaHour`를 추가하고, `/api/config` 호출 시 `otaHour` 매개변수를 쿼리스트링으로 받아 유효 범위를 체크한 뒤 NVS에 최종 저장하게 연동했습니다.
2. **HTML/JS UI 확장:** 민감도 설정 카드 내에 "⏰ Auto-OTA Hour (0-23)" 입력 폼을 신설하고, 대시보드 접속 시 현재 활성화된 설정을 불러와 입력란을 자동으로 초기 채움하도록 했습니다. "Save Settings" 전송 시 해당 설정 값이 주소 파라미터로 함께 빌드되어 저장됩니다.
