# 스마트 박스 강제 오픈 데드락 버그 수정 리포트

## 1. 개요
* **날짜**: 2026-07-23
* **목적**: MQTT 등 외부에서 `forceOpen()` 및 `forceClose()` 명령 호출 시 문이 동작하지 않고 즉시 `STATE_IDLE`로 돌아가는 버그 해결
* **관련 모듈**: `SmartBoxController` FSM (Finite State Machine)

## 2. 문제 원인 분석
사용자로부터 "문이 열려있는 상태에서 IDLE이 지속되고 있으며 강제 오픈 커맨드를 보냈는데 OPEN_START에서 IDLE로 바로 넘어가서 문이 계속 열려있다"는 리포트가 접수되었습니다.

FSM 코드 구조를 분석한 결과, `SmartBoxController::forceOpen()` 호출 시 `STATE_OPEN_START`로 상태가 성공적으로 변경됩니다. 하지만 이전에 발생했던 센서의 데드락(`sensorDeadlockFlag = true`)이나 쿨다운(`isCooldown = true`) 관련 타이머가 여전히 메모리에 잔존해 있는 상태였습니다.

이로 인해 FSM의 메인 루프인 `update()`가 실행될 때, 강제로 `STATE_OPEN_START` -> `STATE_OPENING` 전환 프로세스가 밟히는 도중에 또는 시작도 하기 전에 예외 플래그 검사(예: Sensor Deadlock Timeout, Cooldown block 등)가 트리거되어 즉각적으로 `STATE_IDLE`로 복귀하는 사이드 이펙트가 원인이었습니다.

## 3. 해결 방안
* `forceOpen()` 및 대칭되는 `forceClose()` 메서드가 호출될 때 시스템 상태 강제 전환뿐만 아니라 FSM 내 예외/방어 플래그들도 모두 초기화하여야 합니다.
* 변경 로직:
  1. `std::lock_guard<std::recursive_mutex> lock(dataMutex);` 구문을 사용하여 스레드 안전성 확보
  2. `sensorDeadlockFlag = false;` (데드락 방지 플래그 초기화)
  3. `isCooldown = false;` (쿨다운 대기 초기화)
  4. `deadlockClearStartTime = 0;` (데드락 검사 타이머 초기화)
  5. 이후 `transitionTo(STATE_OPEN_START)` 또는 `transitionTo(STATE_CLOSE_START)` 호출

## 4. 변경된 주요 코드
```cpp
// src/SmartBoxController.cpp

void SmartBoxController::forceOpen() {
    std::lock_guard<std::recursive_mutex> lock(dataMutex);
    if (currentState != STATE_BATTERY_LOW_SHUTDOWN && currentState != STATE_EMERGENCY_STOP) {
        sensorDeadlockFlag = false;
        isCooldown = false;
        deadlockClearStartTime = 0;
        transitionTo(STATE_OPEN_START);
    }
}

void SmartBoxController::forceClose() {
    std::lock_guard<std::recursive_mutex> lock(dataMutex);
    if (currentState != STATE_BATTERY_LOW_SHUTDOWN && currentState != STATE_EMERGENCY_STOP) {
        if (currentState != STATE_IDLE && currentState != STATE_CLOSING && currentState != STATE_CLOSE_START) {
            maintenanceRequested = false;
            sensorDeadlockFlag = false;
            isCooldown = false;
            deadlockClearStartTime = 0;
            transitionTo(STATE_CLOSE_START);
        }
    }
}
```

* PlatformIO 로컬 Native 단위 테스트(`pio test -e native`) 13/13 Pass
* 타이머와 플래그 초기화를 통해 예외 방지 루틴이 수동 조작 커맨드를 블락(Block)하지 않도록 정상 동작함을 확인.
