# ☀️ 태양광 스마트 자동 수거함 통합 사용자 매뉴얼 (스마트홈 개편본)

본 매뉴얼은 태양광 충전 시스템과 초음파 센서를 결합한 스마트 수거함의 안전한 조립, 운영, Home Assistant 스마트홈 앱 연동 및 비상 대응을 위한 가이드입니다.

---

## 1. 시스템 구성 부품

* **메인 제어부:** ESP32-C6 (초음파 제어, 비동기 논블로킹 FSM, MQTTS over TLS & Home Assistant 통합 연동)
* **센서부:** 방수 초음파 센서 (AJ-SR04T), INA219 전압/전류 모니터링 모듈
* **구동부:** 12V 선형 액추에이터 (Linear Actuator), 1채널 릴레이 (메인 12V 전원 차단용), 2채널 릴레이 (모터 정/역회전 방향 제어용)
* **전원부:** 태양광 패널, Solar Charge Controller, 12V 납축전지

---

## 2. 단계별 조립 및 배선 요약

1. **전원부 결선:** 태양광 패널을 충전 컨트롤러에 연결한 후, 12V 납축전지를 컨트롤러 배터리 단자에 연결합니다.
2. **구동부 설치:** 수거함 뚜껑 경첩과 본체 사이에 12V 선형 액추에이터를 유격 없이 단단히 고정합니다.
3. **제어부 배선:** 상세 핀 맵에 따라 릴레이와 초음파 센서, INA219 모듈을 ESP32-C6에 결선합니다.
   * *자세한 회로도 및 결선 정보는 [개발 매뉴얼](DEVELOPMENT_MANUAL.md)을 참조하십시오.*
4. **글리치 차단 점검:** 최초 기동 시 모터가 오동작하여 튀는 것을 방지하기 위해 릴레이의 기본 제어핀 상태는 고임피던스(`INPUT`) 상태로 유지됩니다.

---

## 3. 🚨 핵심 하드웨어 안전 수칙

이 시스템은 고전류 모터와 기계식 릴레이를 제어하므로 아래 수칙을 반드시 엄수해야 합니다.

1. **릴레이 인터락 (Relay Interlock):** 정회전 릴레이(`GPIO 7`)와 역회전 릴레이(`GPIO 8`)를 동시에 `LOW`(ON)로 출력해서는 안 됩니다. 이는 12V 배터리 단락(쇼트)과 화재를 유발할 수 있습니다.
2. **전원 선-차단 후-방향 전환:** 모터 구동 방향을 전환할 때, 무조건 1채널 메인 릴레이(`GPIO 6`)를 먼저 차단(`INPUT` 고임피던스 상태)하고, 최소 100ms 가드 타임을 보낸 후 방향 릴레이를 스위칭해야 합니다. 방향 접점이 완전히 결합된 뒤 다시 100ms 대기를 거쳐 메인 릴레이를 통전합니다.
3. **고임피던스 대기 모드:** 뚜껑이 열려 대기하거나 닫힌 standby 상태에서는 모든 릴레이 제어 핀을 `INPUT`(고임피던스) 상태로 전환하여 누설 전류를 완전히 리셋하고 대기 전력을 0mA로 격리시킵니다.

## 4. Home Assistant 스마트홈 제어 및 자동 등록 (MQTT Integration)

본 시스템은 기기가 켜짐과 동시에 Home Assistant에 클릭 한 번 없이 자동으로 등록되는 **MQTT Auto Discovery** 기능을 표준 지원합니다.

### 📶 4.1. 최초 설정 및 Wi-Fi 변경 (부팅 후 5분 온디맨드 프로비저닝)
장소를 옮기거나 공유기 정보를 새로 변경해야 하는 경우 아래 절차를 따릅니다:
1. 스마트박스의 전원을 켭니다. **전원이 켜진 후 5분 동안 온디맨드 프로비저닝 AP가 활성화**됩니다.
2. 모바일이나 PC의 Wi-Fi 목록에서 **`SmartBox-WiFi`** 신호를 선택하여 연결합니다. (비밀번호: `smartbox_admin` 또는 설정된 AP 비밀번호)
3. 브라우저가 자동으로 열리거나, 주소창에 **`http://192.168.4.1/`**을 입력하면 초경량 Wi-Fi 설정 페이지가 나타납니다.
4. 연결할 공유기의 SSID와 비밀번호를 입력하고 **[Save & Connect]**를 누릅니다.
5. 웹 화면에서 실시간으로 접속 상태가 확인됩니다:
   * 🔄 **`Connecting to...`**: 공유기에 접속 시도 중
   * ✅ **`Connected! IP: 192.168.x.x`**: 접속 성공 및 정상 IP 할당 완료 (MQTT 관제 개시)
   * ❌ **`Connection failed`**: 접속 실패 (비밀번호 오류 또는 SSID 신호 약함)
* *전원 ON 후 5분이 지나면 전력 절감을 위해 AP 전파 송출이 자동으로 중단되며, 기기는 백그라운드 Station 모드로만 동작합니다.*

### 📱 1초 자동 연결 및 제어 방법
1. 스마트박스의 전원을 켜고 Wi-Fi 망 및 MQTT 브로커에 접속하게 합니다.
2. Home Assistant 앱 또는 웹 화면의 **[설정] ➔ [기기 및 서비스] ➔ [MQTT]** 항목으로 이동하면 **`SmartBox`** 기기와 11개의 엔티티가 자동으로 완벽하게 등록되어 있습니다.
3. 별도의 `configuration.yaml` 코딩이나 설정 없이 즉시 대시보드 카드로 만들어 원격 제어할 수 있습니다.

### 🎛️ 지원되는 원격 제어 버튼 및 메트릭 센서
* **원격 제어 버튼 (Buttons):**
  * `[SmartBox] 수거함 강제 열기`: 원격으로 즉시 뚜껑을 개방합니다.
  * `[SmartBox] 수거함 강제 닫기`: 원격으로 뚜껑을 닫습니다.
  * `[SmartBox] 비상 정지 락 해제`: 과전류 보호로 락인된 비상 상태를 안전하게 수동 리셋합니다.
  * `[SmartBox] 펌웨어 강제 업데이트 (OTA)` / `재부팅`: 원격 오버에어 업데이트 및 리셋을 수행합니다.
* **실시간 텔레메트리 센서 (Sensors):**
  * `[SmartBox] 작동 상태`: 실시간 기기 작동 상태 (`IDLE`, `OPENING`, `HOLD` 등) 파악
  * `[SmartBox] 배터리 전압`: 실시간 배터리 전압 (V) 및 방전 그래프 제공
  * `[SmartBox] 최근 개폐 소요 시간` / `피크 전류`: 1회 개폐 시 동작 시간(s) 및 피크 모터 전류(mA) 측정으로 액추에이터 건강 상태 모니터링
  * `[SmartBox] 모션 감지`: 100cm 이내 접근 시 실시간 모션 센서 가동

## 5. 자동 안전 보호 시스템 (FSM States)

시스템은 상태 머신(FSM)을 통해 다음의 안전 가드를 자율적으로 제어합니다.

* **뚜껑 열림 대기 및 자동 닫힘 (`STATE_HOLD`):**
  * 뚜껑이 완전히 열린 후, 사용자가 센서 감지 범위를 벗어나더라도 바로 닫히지 않고 설정된 시간(기본값 10초) 동안 열린 상태를 유지합니다.
* **협착 방지 비상 정지 (`STATE_EMERGENCY_STOP`):**
  * 뚜껑이 열리거나 닫히는 도중 모터 전류가 과전류 임계값(기본 3000mA)을 3회 연속 초과하면 즉시 모터를 비상 정지하고 모든 릴레이를 고임피던스 상태로 고립시킵니다. Home Assistant 앱으로 경고 푸시 알림이 전송됩니다.
* **저전압 배터리 보호 (`STATE_BATTERY_LOW_SHUTDOWN`):**
  * 배터리 전압이 11.3V 이하로 3초 연속 유지될 경우, 최후의 잔여 전력을 공급하여 뚜껑을 강제로 완전 개방한 후 전원을 완전 격리하여 릴레이 소비전력을 0mA로 동결합니다.
* **무선 펌웨어 업데이트 격리 보호 (`STATE_OTA_UPDATING`):**
  * NAS 및 원격 제어로부터 펌웨어 업데이트 요청을 감지하면 즉시 이 모드로 전환되며, 모든 릴레이 전원이 차단된 상태에서 안전하게 플래시 업그레이드를 진행합니다.

## 6. 문제 해결 가이드 (Troubleshooting)

| 현상 | 원인 분석 및 해결 조치 |
| :--- | :--- |
| **뚜껑이 열리지 않고 앱 반응이 없음** | 1. 12V 메인 전원 스위치와 배터리 잔량을 점검하십시오.<br>2. Home Assistant 앱에서 배터리 전압이 11.3V 이하로 떨어져 저전압 셧다운 상태(`Low Battery`)로 고정되어 있는지 확인하십시오. |
| **앱에서 기기가 오프라인으로 표시됨** | 기기의 Wi-Fi 수신 신호 강도(`[SmartBox] 와이파이 신호 강도`) 및 MQTT 브로커 접속 상태를 확인하십시오. 기기 재부팅 시 수초 내로 자동 복구됩니다. |
