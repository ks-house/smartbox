# 스마트 자동 수거함 S/W 설계 및 계획 수립 완료 보고서 (Walkthrough)

본 문서는 스마트 자동 수거함 개발을 본격적으로 수행하기 위한 소프트웨어 아키텍처 설계 및 TDD 계획 문서 수립에 대한 결과물 보고서입니다. 
신규 요구사항인 **무선 네트워크 & SmartThings 연동** 및 **비동기 웹 관리자 대시보드** 기능 설계가 통합 반영되었습니다.

---

## 🛠️ 작성 및 보강된 설계 문서

1. **[implementation_plan.md](implementation_plan.md)**:
   - 하드웨어 의존성 분리 및 TDD 테스팅 환경을 구축하기 위한 **HAL(Hardware Abstraction Layer) 아키텍처**를 설계했습니다.
   - H-브릿지 릴레이 인터락 및 100ms 가드 타임 보장 등의 안전성 소프트웨어 조치를 정의했습니다.
   - PC 로컬 환경에서 테스트 빌드를 수행하기 위해 PlatformIO `native` 플랫폼 설정을 계획에 수립했습니다.
   - 8가지 FSM 상태 변화 조건 및 각 기능별 TDD 시나리오(RED-GREEN-REFACTOR)를 도출했습니다.
   - **[신규]** 비동기 Wi-Fi 연결 및 SmartThings REST API Webhook 전송을 분리하기 위한 백그라운드 큐 및 RTOS 비동기 태스크 연동 설계를 반영했습니다.
   - **[신규]** 비동기 웹서버(`ESPAsyncWebServer`) 및 WebSocket 실시간 모니터링 데이터 연동 구조를 수립했습니다.
   - **[신규]** 감지 거리, HOLD 시간, 과전류 임계값 파라미터를 동적 변경하고 비휘발성 저장소(`Preferences`)와 연동하는 설계 사양을 정의했습니다.
   - **[신규]** 기존 `native` TDD 검증 무결성을 깨뜨리지 않기 위한 이벤트 콜백/옵저버 패턴 격리 설계 및 TDD 6단계(감도 설정 및 이벤트 콜백 리포트 검증)를 추가했습니다.
2. **[task.md](task.md)**:
   - AI 에이전트가 개발 계획서를 참고하여 1단계(환경 설정)부터 12단계(타겟 보드 실물 및 비동기 웹 검증)까지 단계별로 막힘없이 작업을 이행할 수 있도록 구현 TODO 리스트를 구체적으로 수립했습니다. (Phase 1~4 확장 완료)

## 🔍 향후 검증 및 이행 방안

- **단위 테스트 자동화**: 
  다음 구현 에이전트는 [task.md](task.md)에 따라 `native` 테스트 환경에서 Unity test framework를 이용해 각 FSM 로직, 설정 변경, 콜백 동작 및 예외 처리, 하드웨어 안전 인터락을 코드로 증명할 것입니다.
- **실시간 비동기 무선 기능 검증**:
  TDD를 통한 로직 검증 후, 실제 ESP32-C6 하드웨어에 주입하여 웹 UI 실시간 AJAX/WebSocket 수신, 설정 저장 보존 상태 및 스마트싱스 클라우드 상태 리포팅 기능의 종합 테스트가 이뤄질 예정입니다.
